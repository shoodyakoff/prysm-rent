# Правила работы над проектом PRYSM

## 1. Стили и Цвета
- **Централизация цветов**: Все акцентные цвета и градиенты должны храниться в `css/style.css` в секции `:root` в виде CSS-переменных.
- **Использование**: В коде использовать только `var(--variable-name)`, избегать хардкода HEX-значений для акцентных цветов.
- **Пример**:
  ```css
  :root {
      --gradient-optimization: linear-gradient(135deg, #E9D5FF 0%, #C084FC 100%);
      --color-icon-optimization: #9333EA;
  }
  .element { background: var(--gradient-optimization); }
  ```

## 2. Компоненты
- **Повторяющиеся элементы**: Шапка (Header) и Боковая панель (Sidebar) вынесены в отдельные файлы в папке `components/`.
- **Загрузка**: Компоненты загружаются динамически через `js/main.js` в элементы-плейсхолдеры (`#header-placeholder`, `#sidebar-placeholder`).
- **Новые страницы**: При создании новой страницы необходимо добавить эти плейсхолдеры и подключить `js/main.js`.

## 3. Структура Проекта
- `/` (root) - `index.html` (Главная/Дашборд)
- `/css` - Стили (`style.css`)
- `/js` - Скрипты (`main.js`)
- `/components` - HTML-компоненты (`header.html`, `sidebar.html`)
- `/pages` - Директория для дополнительных страниц (например, `/pages/reports/index.html`)
- `/assets` или `/fonts` - Ресурсы

## 4. Технические Ограничения
- **Без Бэкенда**: Проект полностью статический (HTML/CSS/JS).
- **Без API**: Данные должны быть захардкожены в HTML или загружаться из локальных JSON-файлов (если потребуется).
- **Локальный запуск**: Для корректной работы `fetch` (загрузка компонентов) проект желательно запускать через локальный сервер (Live Server).
